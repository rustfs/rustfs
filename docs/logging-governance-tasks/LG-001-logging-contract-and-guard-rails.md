# LG-001 日志契约、脱敏规则、最小 Guard Rails

## 目标

在开始大面积日志改造前，先冻结一套最小但稳定的治理契约：

- 日志事件模型
- 级别语义
- 脱敏规则
- 最小 guard rails

## 范围

- `docs/` 下规范文档
- `crates/obs` 中可能需要的轻量 helper
- 轻量校验脚本或最小测试入口

## 非目标

- 不在本任务里大面积重写 runtime 日志调用点
- 不做复杂自定义 lint 框架
- 不改变任何业务逻辑

## 详细步骤

1. 冻结公共字段命名规范
2. 冻结 `error / warn / info / debug / trace` 语义边界
3. 明确禁止输出的敏感字段
4. 明确默认掩码输出的字段
5. 识别关键目录中新增 `warn!/error!` 的最低结构化要求
6. 约定 request 主日志与后台重试日志的最小模式
7. 为后续批次定义统一验收口径

## 验证方法

- 文档存在且可被后续任务引用
- 至少一处 helper 或测试可以表达脱敏规则
- 后续任务卡不再重复定义自己的日志口径

## 风险与验收标准

### 风险

- 如果规则过重，后续实施成本会过高
- 如果规则过轻，后续日志改造仍会分叉

### 验收

- 有一份单独的日志规范文档
- 有明确敏感字段清单
- 有最小反回退 guard rails 方案
