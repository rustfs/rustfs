# LG-007 反回退验证与收口

## 目标

在前面批次完成后，补上最小但有效的反回退验证，避免新代码重新引入：

- 敏感字段裸露
- 关键路径纯句子型错误日志
- 默认高频噪音

## 范围

- focused tests
- 轻量脚本或 grep-based checks
- 与 `make pre-commit` 兼容的最小入口

## 非目标

- 不做复杂静态分析平台
- 不把所有日志站点都做硬性 lint
- 不为低价值目录增加重验证负担

## 详细步骤

1. 确定必须阻止的回退类型
2. 为脱敏 helper 增加最小测试
3. 为关键 request 日志形态增加 focused 验证
4. 对关键目录新增 `warn!/error!` 的结构化要求，先做轻量检查
5. 补收口文档，说明哪些目录仍属第二轮治理范围

## 验证方法

- 与各批次 focused tests 一起跑
- 必要时增加一个简单脚本在本地或 CI 中复用
- 最终仍以 `make pre-commit` 为准

## 风险与验收标准

### 风险

- 校验过严会阻碍正常开发
- 校验过松则起不到防回退作用

### 验收

- 常见敏感字段回退能被快速发现
- 关键 request / startup / audit / notify 路径不再轻易退回旧写法
- 后续日志治理有稳定的收口手段
