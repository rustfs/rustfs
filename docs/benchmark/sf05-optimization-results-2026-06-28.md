## 分析结论

### 1. SF05 效果显著

SF05（跳过 Inline 数据 IO Planning）对中等文件效果显著：

| 对象大小 | SF05 提升 | 分析 |
|----------|-----------|------|
| 1KiB | +3.7% | 小幅提升，固定开销仍占主导 |
| 4KiB | +4.0% | 小幅提升 |
| 10KiB | **+30.5%** | 显著提升，IO Planning 开销占比降低 |
| 100KiB | **+15.1%** | 明显提升 |
| 1MiB | **+13.8%** | 明显提升 |

**结论**：SF05 对 10KiB-1MiB 对象效果最好，因为这些尺寸的对象在 inline 条件范围内（<=128KB），且 IO Planning 开销占比较为显著。

### 2. 与 main 分支差距缩小

| 对象大小 | SF01-07 vs main | SF01-07+SF05 vs main | 改善 |
|----------|-----------------|----------------------|------|
| 1KiB | -15.6% | -12.5% | +3.1pp |
| 4KiB | -13.9% | -10.4% | +3.5pp |
| 10KiB | -32.0% | -11.3% | **+20.7pp** |
| 100KiB | -23.6% | -12.1% | **+11.5pp** |
| 1MiB | -19.9% | -8.8% | **+11.1pp** |

**结论**：10KiB-1MiB 区间与 main 的差距从 20-32% 缩小到 9-12%，接近持平。

### 3. 与 MinIO 差距

| 对象大小 | 差距倍数 | 分析 |
|----------|----------|------|
| 1KiB | 8.4x | 仍需深度优化 |
| 4KiB | 5.1x | 仍需深度优化 |
| 10KiB | 8.0x | 仍需深度优化 |
| 100KiB | 4.7x | 有改善但仍落后 |
| 1MiB | 3.5x | 有改善但仍落后 |

**结论**：与 MinIO 的差距仍然显著（3.5-8.4x），但 SF05 已将差距缩小了 10-20%。

### 4. 优化效果汇总

| 优化项 | 1KiB | 4KiB | 10KiB | 100KiB | 1MiB |
|--------|------|------|-------|--------|------|
| SF01: Bucket 缓存 | - | - | - | - | - |
| SF02: Inline 快速路径 | ✓ | ✓ | ✓ | ✓ | - |
| SF03: Cache TTL | - | - | - | - | - |
| SF04: 移除 spawn | - | - | - | - | - |
| SF05: 跳过 IO Planning | +3.7% | +4.0% | **+30.5%** | **+15.1%** | **+13.8%** |
| SF06: 条件化 Lifecycle | - | - | - | - | - |
| SF07: 条件化 Metrics | - | - | - | - | - |
| **累计 vs main** | -12.5% | -10.4% | -11.3% | -12.1% | -8.8% |

### 5. 稳定性

所有测试轮次的标准差 < 2%，无 EOF 错误，无功能性失败。优化分支稳定性良好。

## 下一步建议

### 短期

1. **调查 1KiB-4KiB 瓶颈**：这两个尺寸的提升有限，需要进一步分析固定开销
2. **验证 inline 命中率**：确认多少请求实际走了 inline 快速路径
3. **调整 inline 阈值**：当前 128KB 阈值可能需要根据实际数据调整

### 中期

1. **启用 Codec Streaming**：对 1MiB+ 对象有潜在提升
2. **优化 metadata fanout**：减少元数据读取的固定开销
3. **减少锁争用**：分析 RwLock 在高并发下的争用情况

### 长期

1. **实现 O_DIRECT**：减少 page cache 污染
2. **参考 MinIO 实现**：分析 MinIO 在小文件上的优势来源
3. **持续压测**：建立自动化压测流程，跟踪性能回归
